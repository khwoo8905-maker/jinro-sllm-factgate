# 진로상담 sLLM의 환각을 추적하다 — 무엇을 줄였고, 무엇을 못 줄였나
*담백 · 2026-06 · 온프렘 Qwen3.5-4B / 울산 학생 진로상담*


> **⚠️ 먼저 읽을 한계 (헤드라인은 "0%"가 아니라 이것):** 아래 수치(문서밖 추천 0%·유효 100%)는 **합성 평가셋 N=50 한정**이며, **실제 학생 미검증**이다. **존재하는 학과의 부적합 추천(in-vocabulary)과 본문 산문의 사실오류는 미측정** = **실제 안전성 미보증.** 이 글의 핵심은 "성과 0%"가 아니라 **"환각을 정직하게 측정·추적하는 방법"** 이다.

"sLLM 좋다"는 말은 많지만 진짜 적은 환각이다. 진로상담처럼 미성년자에게 직접 닿는 도메인에선 "틀린 추천"이 "모른다"보다 위험하다. 이 글은 성공담이 아니라 **여러 번의 헛다리와 한계까지 공개하는 기록**이다.

## 평가 프로토콜
모델: Qwen3.5-4B + 커리어넷 RAG + LoRA, 온프렘(GB10). 채점·검증: LLM judge(qwen3.6-35b) + KURE 임베딩 근거검증. 평가셋 합성 valid(N=50). 자기평가 한계 있음 — 상대비교·분별검증으로 읽을 것.

## 1. 첫 함정 — 내 65%도 거품
1차 환각 65%로 보였으나 토큰 절단 측정 오염. 토큰 늘려 → 37%. **내 첫 숫자부터 깠다.**

## 2~3. 헛다리 둘 — 데이터 정제·검색개선 (둘 다 효과 없음)
학습데이터 46% 문서밖→정제 재학습 38%(차이 없음). 검색 커버리지: 희망학과 문서有 37% vs 無 40%(차이 없음). 둘 다 원인 아님.

## 4. 게이트도 두 번 틀림
- v1 문자열 매칭: 너무 엄격, 패러프레이즈 못 알아봐 멀쩡한 답 폴백.
- v2 문서전체 의미유사: 너무 느슨, 무관 학과 통과.
- v3 학과명집합 의미유사: 분별은 되나 **블록 단위라 옆에 붙은 문서밖 학과를 못 거름**(잔존 16~20%).
- **결정적 교훈:** 게이트가 "추천한 학과"와 "그냥 언급된 학과(학생 희망·과목명)"를 구분해야 한다.

## 5. 해결 — 추천-단위 grounding 게이트
추천 헤더/"추천 학과:" 줄의 학과만 근거검증(정확매칭 + KURE 의미유사도≥0.82). 문서밖을 추천한 블록만 드롭, 부수 언급은 보존.

## 측정 결과 (제품 파이프라인, N=50)
- 모델이 **실제 추천한 학과: 135건 중 132건(98%)이 이미 근거 있음.** (앞선 "18% 할루"는 대부분 추천이 아닌 부수 언급이었음)
- 게이트가 **문서밖 추천 3건(2%)을 정확히 드롭.**
- 결과: **문서밖 추천 0%, 폴백 0%, 유효 100%.**

## Known Failure Modes (못 잡는 것)
- 존재하는 학과를 **부적합하게** 추천(in-vocabulary 오류) — 미측정.
- 추천 아닌 본문 산문의 사실오류, 부수 학과 언급.
- 합성 평가셋(N=50) 기반 — 실제 학생·대규모 분포 입증은 별도.

## 교훈
평가 노이즈부터 의심 · 직관적 fix를 싸게 falsify(데이터·검색·게이트 모두 첫 시도는 틀림) · 환각은 학습 아닌 추론 단 grounding으로 · "추천"과 "언급"을 구분해야 진짜로 잡힌다 · 무명에겐 0% 자랑보다 실패·한계 공개가 더 강한 자산.
